# 网络综合实验平台 - 考试系统说明

## 📋 总览

本考试系统为**七年级信息科技第二单元《直播网络我来建》**设计，包含三个难度等级的网络配置考试，全面考察学生对路由器、DNS、Web服务器等网络设备的配置和故障排查能力。

**适用版本**：网络综合实验平台 v2.8.3
**考试时长**：40分钟/场
**总分**：40分/场
**及格线**：24分（60%）

## 🎯 难度等级说明

### B级 - 基础网络配置

**考试目标**：完成基本的两个局域网互联，配置DNS和Web服务器

**拓扑结构**：
```
PC1 --- R1 --- R2 --- Web1
                 \
                  DNS1
```

**设备数量**：5台（2路由器 + 1PC + 1Web + 1DNS）

**配置特点**：
- ✅ 所有设备使用192.168.x.x网段，便于学生理解
- ✅ 所有配置项为空，学生从零开始配置
- ✅ 拓扑结构简单线性，无复杂路径

**核心考点**：
1. PC网关和DNS服务器配置
2. 路由器路由表配置（双向路由）
3. DNS服务器域名记录配置
4. Web服务器端口、域名、内容配置
5. Ping和浏览器测试

**适用场景**：
- 七年级第二单元期末考试（基础要求）
- 第8-10课单元复习测试
- 补考或基础薄弱学生测试

**预计通过率**：70-80%

---

### A级 - 路由优化与多路径选择

**考试目标**：理解路由度量（Metric），配置最优路径

**拓扑结构**：
```
           R2 (中转)
          /  \
PC1 --- R1    R3 --- Web1
         \    /       |
          \--/       DNS1
       (直连近路)
```

**设备数量**：6台（3路由器 + 1PC + 1Web + 1DNS）

**配置特点**：
- ✅ 非对称拓扑，R1到R3有两条路径（远路和近路）
- ✅ R2和R3已部分配置（提供提示）
- ✅ R1需要学生完全配置，并理解Metric优先级
- ✅ 混合网段（192.168.x.x + 10.0.x.x）

**核心考点**：
1. 拓扑结构分析（识别多条路径）
2. 路由表配置（同一目标网络的多条路由）
3. Metric度量值设置（近路Metric=1，远路Metric=2）
4. 连通测试路径验证（确认选择了最优路径）
5. 浏览器测试

**适用场景**：
- 七年级第二单元期末考试（优秀学生）
- 网络社团或兴趣班考核
- 八年级入学摸底测试

**预计通过率**：40-50%

**关键难点**：
- 理解Metric的含义（跳数/优先级）
- 配置两条指向同一目标网络的路由
- 使用连通测试验证路径选择

---

### A+级 - 网络故障诊断与排除

**考试目标**：诊断并修复5个预设的配置错误

**拓扑结构**：
```
PC1 --- R1 --- R3 --- R4 --- Web1
         |      |      |
        R2 -----+      +---- Web2
         |             |
        PC2           DNS1
```

**设备数量**：9台（4路由器 + 2PC + 2Web + 1DNS）

**配置特点**：
- ⚠️ 包含5个故意设置的错误
- ⚠️ 错误涵盖网络层、应用层和配置细节
- ⚠️ 需要使用连通测试、浏览器等多种工具诊断
- ✅ 大部分设备已配置，学生只需修复错误

**预设错误清单**：

| 错误编号 | 错误类型 | 错误位置 | 诊断难度 |
|---------|---------|---------|---------|
| 错误1 | 路由环路 | R1和R3路由表 | ⭐⭐⭐⭐ 困难 |
| 错误2 | 网关配置 | PC1 | ⭐ 简单 |
| 错误3 | 端口不匹配 | Web1 | ⭐⭐ 中等 |
| 错误4 | DNS记录 | DNS1 | ⭐⭐ 中等 |
| 错误5 | 接口名称 | R1路由表 | ⭐⭐⭐ 中等偏难 |

**核心考点**：
1. 逐层故障诊断（网络层→应用层）
2. 路由环路识别与修复（连通测试工具使用）
3. 配置细节检查（接口名称、IP地址、端口）
4. 多工具综合运用（连通测试、浏览器）
5. 系统性故障排查思维

**适用场景**：
- 网络技术竞赛选拔
- 八年级拔尖学生测试
- 信息科技特长生考核
- 教师培训和示范教学

**预计通过率**：20-30%

**关键难点**：
- 发现并理解路由环路（最大难点）
- 注意到接口名称等配置细节
- 全面检查不遗漏任何错误（5个错误都要找到）

### 批改要点

#### B级批改重点
✅ **必查项**（80%分值）：
- PC1网关和DNS配置
- R1和R2路由表（目标网络、下一站、接口）
- DNS记录和Web服务器域名一致性

⚠️ **易错点**：
- 路由表的"下一站"填成自己的IP
- 路由表的"接口"填错方向
- DNS记录的IP与Web服务器IP不一致

#### A级批改重点
✅ **必查项**（80%分值）：
- R1路由表有两条指向192.168.3.0的路由
- 近路径Metric < 远路径Metric
- 连通测试显示走近路径（不经过R2）

⚠️ **易错点**：
- 只配置了一条路由
- Metric配置反了（近路大于远路）
- 路由表下一站或接口填错

#### A+级批改重点
✅ **必查项**（100%分值）：
- 5个错误的修复情况（每个错误8分）
- 路由环路是否修复（最关键，10分）
- 全网连通性测试（5分）

⚠️ **给分技巧**：
- 修复3个以上错误且网络基本连通：及格（24分）
- 修复4个错误：良好（30-35分）
- 修复5个错误且测试全通过：优秀（36-40分）

## 🔧 故障排查指南（教师参考）

### 学生常见求助问题

#### 问题1："我配置了路由表，但Ping还是不通"
**诊断步骤**：
1. 检查路由表的"下一站"是否正确（对方路由器的骨干网IP）
2. 检查路由表的"接口"是否正确（自己路由器连接对方的接口名）
3. 检查是否配置了双向路由（去程和回程）
4. 使用连通测试查看数据包走到哪里失败

**教学提示**：
- 引导学生使用"接口管理器"查看接口名称和IP
- 强调"下一站"是对方路由器的IP，不是自己的IP

---

#### 问题2："浏览器显示DNS解析失败"
**诊断步骤**：
1. 检查PC的DNS服务器是否配置
2. 检查PC能否Ping通DNS服务器
3. 检查DNS服务器是否有对应的域名记录
4. 检查域名拼写是否正确

**教学提示**：
- DNS解析失败通常是PC未配置DNS服务器地址
- 或者DNS记录不存在

---

#### 问题3："浏览器显示端口错误"
**诊断步骤**：
1. 检查Web服务器监听端口（应该是80）
2. 检查浏览器访问的端口（默认80）
3. 如果Web服务器端口不是80，URL需要明确指定端口

**教学提示**：
- HTTP默认端口是80
- 如果服务器监听8080，访问时需要写成 domain:8080

---

#### 问题4："连通测试显示数据包来回转发"（A+级）
**诊断步骤**：
1. 这是典型的路由环路现象
2. 检查路由表中两个路由器是否互相指向对方
3. 修改其中一个路由器的路由表，打破环路

**教学提示**：
- 路由环路是A+级的核心难点
- 需要分析两个路由器的路由表配置

## 📞 技术支持

### 常见问题

**Q1: JSON文件导入后设备位置混乱怎么办？**
A: 可以手动拖拽设备调整位置，不影响配置和测试。

**Q2: 学生删除并重新连接路由器，骨干网网段变化了怎么办？**
A: 骨干网由系统自动分配，网段号可能变化，但不会影响拓扑逻辑。批改时需要根据实际网段调整检查标准。

**Q3: 如何快速清空学生配置，重新开始考试？**
A: 重新导入JSON文件，所有配置会恢复到初始状态。

**Q4: 考试时学生电脑无法访问教师机怎么办？**
A: 检查：
1. 教师机和学生机在同一局域网
2. 教师机防火墙允许端口5173
3. 使用`npm run dev`启动开发服务器（自动监听0.0.0.0）

**Q5: 考试时平台崩溃或数据丢失怎么办？**
A: 平台使用LocalStorage自动保存，刷新页面即可恢复。建议学生定期点击"保存"（使用场景保存功能）。
